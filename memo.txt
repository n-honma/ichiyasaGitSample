Pull Request
    - 作成したブランチの取り込みを依頼するとき
    ・変更を加えた人以外が内容をチェックする　→レビュー

プッシュ
    - 変更をリモートリポジトリに反映する
        ローカルリポジトリで作ったブランチをリモートリポジトリにも反映できる


1. ブランチを作成
2. 作ったブランチにチェックアウト
（git checkout -b :新しくブランチを作って同時にチェックアウト）
3. 対象のソースを変更
4. ローカルリポジトリにコミット(ここまで自分の中でやる)
5. プッシュ
6. プルリクエスト
7. マージ
8. プル/フェッチ

プル/フェッチ
    - リモートリポジトリの内容を使用中のローカルリポジトリに取り込む
    違い:ワークツリーの設定まで変更されるかどうか

作業ごとにブランチをつくる

１　スピーカー情報更新
　　一人目の画像追加は後回し
２　タイムテーブルの変更
　→masterブランチへマージ（タイムテーブル）
３　スピーカー情報更新
　→masterブランチへマージ（スピーカー）
    git merge 取得したいブランチ

ブランチもディレクトリと同じで作る場所に注意
    一度masterに戻ってから作るのがよい

コンフリクト:競合
